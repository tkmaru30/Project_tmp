# 🚀 Slack連携開発ワークスペース

Slackから自然言語で指示するだけで、タスク管理・アイデア管理・文書作成・Webアプリ開発が行える統合開発環境です。

## 📋 プロジェクト概要

### 🎯 主な特徴
- **🤖 Slack完全連携**: チャットで全ての操作が可能
- **📝 統合文書管理**: タスク・アイデア・議事録・メモの一元管理
- **⚡ 自動化ワークフロー**: 文書間の自動連携と変換機能
- **🌐 Webアプリ作成**: 指示するだけでアプリを自動生成
- **🇯🇵 完全日本語対応**: 全ての機能を日本語で利用可能

## 💬 Slackからの指示方法

このワークスペースの全ての機能は、Slackからの自然言語指示で操作できます。以下に主な指示方法を説明します。

### 📋 タスク管理

#### タスクの追加
```

# 詳細指定でのタスク追加
@Cursor 緊急タスク「セキュリティパッチの適用」を優先度Criticalで追加してください

# 担当者・期限指定

#### タスクの確認・検索
```

## 🔄 自動化ワークフロー

### 文書間連携
- **アイデア → タスク**: アイデアを実行可能タスクに自動分解
- **議事録 → タスク**: アクションアイテムを自動でタスク化
- **メモ → アイデア/タスク**: メモ内容に応じて適切な形式に変換

### GitHub Actions 自動マージ
このプロジェクトでは、GitHub Actionsを使用してプルリクエストの自動マージ機能を実装しています。

#### 主な機能
- **ラベルベースの自動マージ**: `auto-merge`または`ready-to-merge`ラベルで自動マージをトリガー
- **レビュー要求の確認**: 必要なレビュー数が満たされているかを自動チェック
- **ステータスチェックの確認**: すべてのCI/CDチェックが成功しているかを確認
- **セキュリティ機能**: 機密ファイルの変更検出やブランチ保護の確認
- **WIP（作業中）対応**: 作業完了時の自動マージとラベル管理

#### 使用方法
1. プルリクエストを作成
2. 自動マージ用ラベル（`auto-merge`または`ready-to-merge`）を追加
3. 必要なレビューを取得
4. すべてのチェックが成功するのを待つ
5. 自動的にマージされる

詳細な設定と使用方法については、[📁 .github/AUTO_MERGE_README.md](./.github/AUTO_MERGE_README.md)を参照してください。

### 定期自動実行
```
# 日次実行（毎朝9:00）
@Cursor おはようございます！今日のタスク一覧と緊急事項を教えてください

# 週次実行（毎週月曜日10:00）
@Cursor 先週の完了タスクとアイデアの追加状況をレポートしてください

# 月次実行（毎月1日11:00）
@Cursor 先月の生産性レポートを作成してください
```
