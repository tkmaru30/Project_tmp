# Tasks Directory

This directory contains all task-related documentation and management files.

## Structure

- `01_backlog/` - Tasks that are planned but not yet started
- `02_active/` - Tasks currently being worked on
- `03_completed/` - Tasks that have been finished
- `09_templates/` - Templates for creating new tasks

## Task Management

### Creating Tasks
1. Use templates from `09_templates/` directory
2. Include clear acceptance criteria
3. Add relevant labels and priorities
4. Assign to appropriate team members

### Task Lifecycle
1. Tasks start in `01_backlog/`
2. Move to `02_active/` when work begins
3. Move to `03_completed/` when finished
4. Archive completed tasks regularly

### Naming Convention
- Use descriptive filenames
- Include task ID or reference number
- Add date prefix for chronological ordering
- Use consistent format: `YYYY-MM-DD_task-name.md`
